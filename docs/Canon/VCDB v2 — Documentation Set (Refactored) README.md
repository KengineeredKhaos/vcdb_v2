# VCDB v2 — Documentation Set (Refactored)

This mini set reorganizes the current notes into purpose-based documents.

## Files

1. **VCDB v2 — Ethos & Canon Invariants (Refactored).md**
   
   - non-negotiable canon
   - architecture rules
   - ownership rules
   - privacy, ledger, and money-truth rules

2. **VCDB v2 — System ShapeRelationships & Money Flow Guide.md**
   
   - plain-English explanation of the application shape
   - slice responsibilities
   - funding pipeline and execution story
   - successor-friendly mental model

3. **VCDB v2 — Access, Admin & Permissions Plan.md**
   
   - access structure
   - Admin mission and boundaries
   - slice surface matrix
   - function/action planning template

4. **VCDB v2 — Project-wide TODO (Curated).md**
   
   - grouped backlog
   - planning items separated from canon

## Why the split

The source material mixed together:

- canon
- explanation
- permissions planning
- Admin feature notes
- finance rubric notes
- TODO backlog

This split keeps the canon short enough to enforce, while moving explanatory
and planning material into documents that can evolve without destabilizing the
Ethos file.
