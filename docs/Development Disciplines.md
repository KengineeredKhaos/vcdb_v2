# Development Disciplines



- Frontend
- Backend
- UI
- UX
- System architecture
- Domain modeling
- Workflow or business process design
- DevOps / operations
- Security
- Product design



**Frontend development**  
What the operator sees and clicks. Templates, pages, forms, CSS, layout, interaction feel.

**Backend development**  
The application’s working guts. Routes, services, business rules, database writes, permissions, validation.

**UI design**  
How screens, buttons, menus, spacing, and visual hierarchy are arranged.

**UX design**  
How the workflow feels to the human using it. What they expect, what confuses them, what helps them recover from interruption.

**System architecture**  
The big-picture structural design. Slice boundaries, contracts, data ownership, how parts talk to each other, what belongs where.

**Domain modeling**  
Turning real-world business truth into software concepts. Customer, Sponsor, Resource, Contact, Officer, Intake, Referral, Ledger event, and so on.

**Business process design** or **workflow design**  
How the real work is supposed to happen operationally. Who does what first, what happens when interrupted, what gets reviewed, what gets escalated.

**DevOps**  
How the app gets deployed, configured, updated, backed up, and kept running.

**Operations** or **IT operations**  
The day-to-day running environment: kiosks, user accounts, password resets, printer issues, backups, local network realities.

**Security engineering**  
Authentication, authorization, session handling, auditability, least privilege, safe data handling.

**Information architecture**  
How information is grouped and navigated. What goes in navbar vs homepage vs slice page. What is global vs local.
